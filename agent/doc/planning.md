# 规划策略（Planning Strategy）

## 概述

本 Agent 采用 **"分析 → 补全 → 规划 → 确认 → 执行"** 的五阶段规划策略，基于 LangGraph 状态图驱动。

## 核心流程

```
用户自然语言输入
       │
       ▼
 ┌─────────────┐
 │  1. 意图分析  │  ← LLM 提取意图、场景、结构化信息
 └──────┬──────┘
        │
   ┌────┴─────┬──────────┬──────────┐
   ▼          ▼          ▼          ▼
 偏好设置    打断处理    信息不足    正常规划
   │          │          │          │
   │          ▼          ▼          ▼
   │     重新分析    追问补充    选择 Skill
   │          │          │          │
   │          └────┬─────┘          │
   │               ▼                │
   │          生成方案 ◄─────────────┘
   │               │
   │               ▼
   │          等待确认
   │               │
   │          用户确认
   │               │
   ▼               ▼
 END          执行下单 → 格式化输出 → END
```

## 阶段详解

### 1. 意图分析（analyze_input）

**目标**：理解用户想做什么。

**输入**：用户自然语言 + 对话历史 + 用户偏好

**输出**：结构化 JSON，包含：
- `intent`：意图类型（plan/preference/confirm/interrupt）
- `scenario`：场景（family/friends/couple/solo）
- `extracted_info`：提取的关键信息（人数、年龄、预算等）
- `missing_info`：缺失的必要信息列表

**场景判断逻辑**：
| 关键词 | 场景 |
|--------|------|
| 孩子/老婆/家人/亲子 | family |
| 朋友/哥们/闺蜜/多人 | friends |
| 对象/女朋友/男朋友/约会 | couple |
| 无明确同伴信息 | solo |

### 2. 信息补全（gather_info）

**目标**：在信息不足时自然地追问用户。

**策略**：
- 一次最多问 2 个问题，避免让用户感到被盘问
- 基于已有信息给出合理建议
- 不重复询问偏好中已有的信息
- 语气友好自然，像朋友聊天

**触发条件**：`missing_info` 非空

### 3. 方案规划（generate_plan）

**目标**：生成可执行的完整活动方案。

**策略**：

1. **数据筛选**：从 mock 数据中按场景标签过滤，取评分前 5 的候选
2. **偏好融合**：将用户偏好（饮食限制、距离要求等）融入提示词
3. **打断融合**：如果用户中途加入新信息（如"我老婆在减肥"），一并考虑
4. **结构化输出**：要求 LLM 返回严格的 JSON 格式，包含时间线、费用、建议

**方案约束**：
- 时间安排合理，考虑交通时间
- 活动和用餐穿插，不要太紧凑
- 距离不要太远，尽量在同一区域
- 餐厅排队时间不要太长
- 亲子场景优先有儿童设施的场所
- 朋友场景活动要适合多人参与

### 4. 用户确认（wait_confirmation）

**目标**：让用户审核方案后再执行。

**策略**：
- 方案生成后进入等待状态
- 用户说"确认"/"好的"/"执行"等 → 触发下单
- 用户说其他内容 → 视为打断，重新调整方案

### 5. 自动执行（execute_plan）

**目标**：用户确认后自动完成所有预订。

**执行顺序**：
1. 遍历方案 timeline，对每个 activity 调用 `book_activity()`
2. 对每个 meal 调用 `book_restaurant()`（亲子场景自动加儿童椅）
3. 处理 extra_suggestions，下单蛋糕/鲜花并配送到餐厅
4. 所有订单生成订单号，持久化到 `data/orders.json`

## 偏好机制

### 被动学习
- 用户在对话中自然提到的信息（如"我老婆在减肥"）会被自动提取为偏好
- 偏好持久化到 `data/preferences.json`，下次规划自动参考

### 主动设置
- 用户通过 `/偏好` 命令主动设置偏好
- 支持：喜欢的菜系、不喜欢的菜系、饮食限制、活动类型、最远距离、预算、同行人

## 打断机制

用户可以在方案讨论中随时加入新信息：

```
用户: 今天下午带孩子出去玩
Agent: [生成方案]
用户: 对了，我老婆在减肥
Agent: [分析打断 → 重新生成方案]
```

**打断处理流程**：
1. LLM 分析新信息的影响程度（low/medium/high）
2. 更新分析结果中的 extracted_info
3. 自动提取偏好（如减肥→饮食限制）
4. 用更新后的信息重新生成方案
